## Session 1 — Modular refactor + richer metrics

### Done
- Split 1449-line `src/main.rs` into modules: `main.rs`, `app.rs`, `snapshot.rs`, `ai.rs`, `tray.rs`, `inspector.rs` (+ `inspector.html`), `services.rs`, `util.rs`.
- Introduced `Sampler` in `snapshot.rs` owning `System`, `Users`, `Networks`, and localhost/sample timing state.
- Added GPU% via `ioreg -rc IOAccelerator` parse, per-core CPU (`cores` in snapshot), load average, network ↓/↑ bytes/s, disk read/write bytes/s (summed from per-process `disk_usage`).
- Tray menu now surfaces CPU, RAM, Swap, GPU, Load, Net, Disk, AI at a glance; tray title reformatted to `23%C 52%R 14%G AI2`.
- Removed the pending-kill / confirm submenu dance. Tray submenu "Kill process" fires immediately; inspector keeps a JS `confirm()` guard for a light confirmation.
- Inspector UI rebuilt: 6-tile stat grid with fill bars, per-core mini bar grid, heat-dot on process CPU, scrollable process table, AI workloads / token checker / provider keys panels retained.
- README updated to reflect new tray title format, metrics surfaced, and one-click kill flow.

### Decisions
- **CPU accuracy fix via warm-up in `Sampler::new`:** one `refresh_cpu_usage()` + 220ms sleep before returning the sampler. sysinfo needs two refreshes ≥200ms apart for meaningful CPU deltas; without this the first snapshot read ~0% even on a busy system (user reported Activity Monitor at ~30% vs tray at ~1%).
- **Disk I/O sourced from per-process `disk_usage` sum** rather than from `Disks`. In sysinfo 0.37 the `Disks` API does not expose a byte-rate counter; per-process delta bytes since last refresh, divided by elapsed wall-clock, gives a usable total rate without extra deps. Dropped the `Disks` field after verifying it wasn't needed.
- **GPU via `ioreg -rc IOAccelerator`** (no root, no `powermetrics`). Parses `"Device Utilization %"` line from the PerformanceStatistics dict. Returns `None` on machines where the key is missing — UI renders `n/a`.
- **One HTML file `include_str!`'d** instead of templated from Rust strings — easier to iterate on CSS/JS.
- **Kept `keyring::use_native_store(false)` call** as-is; existing flow depends on it and changing it is out of scope for this session.

### Blockers / Next
- Runtime not yet smoke-tested — only `cargo build --release` verified (46s, clean). Need to reinstall via `scripts/install.sh` + `launchctl unload/load` to confirm metrics render correctly.
- GPU parsing is Apple Silicon-oriented; behaviour on Intel Macs untested.
- First sample after launch will still show near-zero network/disk rates (no prior sample to diff against); acceptable, but could display "—" on first tick if needed.
- If `global_cpu_usage()` still under-reports after the warm-up fix, fallback plan: average `cpus().iter().map(|c| c.cpu_usage())` manually and compare.
